<a name="bug-admin-001"></a>
## BUG-ADMIN-001: Статус заявки не обновляется без перезагрузки страницы

| Field        | Details |
|--------------|---------|
| **Module**   | Управление заявками на эвакуатор |
| **Severity** | Major |
| **Priority** | High |
| **Status**   | Open |

### Steps to Reproduce
1. Открыть список заявок на эвакуатор
2. Параллельно (другой вкладкой или коллегой) изменить статус заявки на «Передана в эвакуацию»
3. Вернуться к списку — не перезагружая страницу
4. Найти эту заявку в списке

### Expected Result
Список обновляется автоматически в реальном времени (WebSocket / polling) — статус «Передана в эвакуацию» виден без ручного обновления.

### Actual Result
Статус заявки в списке остаётся «Новая». Сотрудник не видит изменений и может повторно отправить уже обработанную заявку.

> ⚠️ **Impact:** Критично при работе нескольких операторов одновременно — высокий риск дублирования обработки заявки.

<a name="bug-admin-002"></a>
## BUG-ADMIN-002: Кнопка «Отправить в отдел эвакуации» активна для уже переданных заявок

| Field        | Details |
|--------------|---------|
| **Module**   | Управление заявками на эвакуатор |
| **Severity** | Major |
| **Priority** | High |
| **Status**   | Open |

### Steps to Reproduce
1. Найти заявку со статусом «Передана в эвакуацию»
2. Открыть карточку заявки
3. Проверить состояние кнопки «Отправить в отдел эвакуации»
4. Нажать на кнопку

### Expected Result
Кнопка недоступна (`disabled`) или скрыта — повторная отправка заявки невозможна.

### Actual Result
Кнопка активна. При нажатии заявка отправляется повторно в отдел эвакуации.

> ⚠️ **Impact:** Дублирование заявок в отделе эвакуации, путаница при обработке.

## BUG-ADMIN-003: Черновик сообщения пропадает при переключении между диалогами

| Field        | Details |
|--------------|---------|
| **Module**   | Чат поддержки |
| **Severity** | Minor |
| **Priority** | Medium |
| **Status**   | Open |
| **Browser**  | Firefox 125 |

### Steps to Reproduce
1. Открыть диалог с пользователем А
2. Ввести текст в поле ввода — **не отправлять**
3. Переключиться на диалог с пользователем Б
4. Вернуться к диалогу с пользователем А

### Expected Result
Введённый текст сохранён как черновик и отображается в поле ввода при возврате к диалогу.

### Actual Result
Поле ввода пустое — набранный текст утерян.

## BUG-ADMIN-004: Счётчик непрочитанных сообщений не сбрасывается после прочтения

| Field        | Details |
|--------------|---------|
| **Module**   | Чат поддержки |
| **Severity** | Minor |
| **Priority** | Medium |
| **Status**   | Open |

### Steps to Reproduce
1. Дождаться нового входящего сообщения (счётчик = 1)
2. Открыть диалог и прочитать сообщение
3. Вернуться к списку диалогов

### Expected Result
Счётчик непрочитанных сообщений сбрасывается до 0 после открытия диалога.

### Actual Result
Счётчик остаётся на «1» — не обнуляется автоматически.
